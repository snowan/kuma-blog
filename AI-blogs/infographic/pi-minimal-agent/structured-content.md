# Pi Architecture Infographic - Structured Content

## Title
Pi: The Minimal Agent Architecture

## Learning Objectives
- Understand Pi's 4-tool minimal core
- Learn how the extension system works
- See how tree sessions enable branching workflows

## Sections

### Section 1: Central Hub - Pi Core
**Key Concept**: Pi's Minimal Core
**Content**: Pi is a coding agent with only 4 core tools: Read, Write, Edit, Bash
**Visual Element**: Central hub with "Pi Core" label
**Text Labels**: "Pi Core", "Shortest System Prompt"

### Section 2: Spoke 1 - Read Tool
**Key Concept**: Read
**Content**: Read file contents and documentation
**Visual Element**: Book/document icon
**Text Labels**: "Read", "Files & Docs"

### Section 3: Spoke 2 - Write Tool
**Key Concept**: Write
**Content**: Write to files
**Visual Element**: Pencil/pen icon  
**Text Labels**: "Write", "Create Files"

### Section 4: Spoke 3 - Edit Tool
**Key Concept**: Edit
**Content**: Modify existing files
**Visual Element**: Edit/scissors icon
**Text Labels**: "Edit", "Modify Code"

### Section 5: Spoke 4 - Bash Tool
**Key Concept**: Bash
**Content**: Execute shell commands, run any CLI
**Visual Element**: Terminal icon
**Text Labels**: "Bash", "Execute Commands"

### Section 6: Extension Arm - Self-Extension
**Key Concept**: Extension System
**Content**: Agent extends itself by writing code, hot reloading
**Visual Element**: Connected nodes branching out
**Text Labels**: "Extensions", "Self-Extending"

### Section 7: Tree Sessions
**Key Concept**: Tree-Based Sessions
**Content**: Branch for side-quests, merge back without context pollution
**Visual Element**: Tree branching diagram
**Text Labels**: "Tree Sessions", "Branch & Merge"

## Design Instructions
- Use hub-spoke layout with Pi Core in center
- Show 4 main tools as primary spokes
- Add extension system as secondary arm
- Tree sessions as decorative element
- Hand-drawn craft paper style
- Warm pastel colors
